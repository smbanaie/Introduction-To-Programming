# فرمت باینری: زبان کامپیوترها

## چرا باینری؟ واقعیت الکتریکی

کامپیوترها ماشین‌های الکتریکی هستند. آنها فقط دو حالت را درک می‌کنند: **روشن** (جریان برق) و **خاموش** (عدم جریان برق). این ویژگی الکتریکی اساسی تعیین می‌کند که چرا کامپیوترها از باینری استفاده می‌کنند.

## باینری چیست؟

**باینری** یک سیستم عددی است که فقط دو رقم دارد: ۰ و ۱.

### مقایسه با اعشار
| سیستم | پایه | رقم‌ها | مثال |
|--------|------|--------|-------|
| **اعشار** | ۱۰ | ۰-۹ | ۴۲ |
| **باینری** | ۲ | ۰-۱ | ۱۰۱۰۱۰ |

### تبدیل بین سیستم‌ها
```
اعشار ۵ = باینری ۱۰۱
اعشار ۱۰ = باینری ۱۰۱۰
اعشار ۱۵ = باینری ۱۱۱۱
```

## چرا کامپیوترها به باینری نیاز دارند

### ۱. **سادگی الکتریکی**
- **ترانزیستورها**: سوئیچ‌های الکترونیکی که روشن یا خاموش هستند
- **قابل اعتماد**: فقط دو حالت به معنای خطاهای کمتر
- **سریع**: مدارهای ساده برای عملیات پایه

### ۲. **منطق دیجیتال**
باینری امکان **منطق بولی** را فراهم می‌کند:
- **AND**: هر دو ورودی باید ۱ باشند
- **OR**: حداقل یک ورودی باید ۱ باشد
- **NOT**: ۰ را به ۱ و ۱ را به ۰ تبدیل می‌کند

### ۳. **قابلیت توسعه**
- **بیت‌ها ترکیب می‌شوند**: ۸ بیت = ۱ بایت (۲۵۶ ترکیب)
- **رشد نمایی**: بیت‌های بیشتر = امکانات بیشتر
- **انعطاف‌پذیر**: می‌تواند هر نوع داده‌ای را نمایش دهد

## نمایش داده‌های باینری

### اعداد
```
اعشار: ۴۲
باینری:  ۰۰۱۰۱۰۱۰
         ││││││││
         ۳۲+۸+۲=۴۲
```

### متن (ASCII)
```
'A' = ۰۱۰۰۰۰۰۱ (اعشار ۶۵)
'B' = ۰۱۰۰۰۰۱۰ (اعشار ۶۶)
'سلام' = ۰۱۰۱۰۰۰۰ 01100101 01101100 01100001 01101101
```

### رنگ‌ها (RGB)
```
قرمز خالص: ۱۱۱۱۱۱۱۱ ۰۰۰۰۰۰۰۰ ۰۰۰۰۰۰۰۰
          (قرمز ۲۵۵، سبز ۰، آبی ۰)
```

### تصویر و ویدیو
- **پیکسل‌ها**: هر پیکسل مقادیر RGB باینری است
- **فشرده‌سازی**: الگوریتم‌ها اندازه باینری را کاهش می‌دهند
- **فرمت‌ها**: همه JPEG، PNG، MP4 به داده‌های باینری تبدیل می‌شوند

## عملیات باینری

### ریاضی پایه
```
جمع:   ۵ + ۳
      ۰۱۰۱
    + ۰۰۱۱
    -------
      ۱۰۰۰ (۸)

تفریق: مشابه اعشار با قرض گرفتن
ضرب: جمع مکرر
تقسیم: تفریق مکرر
```

### عملیات منطقی
```
AND: ۱ & ۱ = ۱، ۱ & ۰ = ۰، ۰ & ۰ = ۰
OR:  ۱ | ۱ = ۱، ۱ | ۰ = ۱، ۰ | ۰ = ۰
XOR: ۱ ^ ۱ = ۰، ۱ ^ ۰ = ۱، ۰ ^ ۰ = ۰
NOT: ~۱ = ۰، ~۰ = ۱
```

## واحدهای ذخیره داده

### بیت (رقم باینری)
- کوچکترین واحد: ۰ یا ۱
- یک حالت الکتریکی را نشان می‌دهد

### بایت (۸ بیت)
- واحد استاندارد: ۲۵۶ مقدار ممکن (۰-۲۵۵)
- برای کاراکترها، اعداد کوچک استفاده می‌شود

### واحدهای بزرگ‌تر
```
۱ کیلوبایت (KB) = ۱,۰۰۰ بایت (در واقع ۱,۰۲۴)
۱ مگابایت (MB) = ۱,۰۰۰ KB
۱ گیگابایت (GB) = ۱,۰۰۰ MB
۱ ترابایت (TB) = ۱,۰۰۰ GB
```

## باینری در برنامه‌نویسی

### کد ماشین
برنامه‌ها به دستورالعمل‌های باینری تبدیل می‌شوند:
```
اسمبلی: ADD R1, R2, R3
باینری:   ۰۰۰۰۰۱ ۰۰۰۱۰ ۰۰۰۱۱ ۰۰۰۰۰ ۱۰۰۰۰۰
```

### انواع داده
انواع داده‌های مختلف فرمت باینری متفاوتی نیاز دارند:
- **صحیح**: نمایش مستقیم باینری
- **شناور**: فرمت خاص (علامت + توان + مانتیس)
- **متن**: کدگذاری کاراکتر (ASCII، Unicode)

### فرمت فایل
همه فایل‌ها در سطح پایین باینری هستند:
- **فایل‌های متنی**: کاراکترهای کدگذاری شده باینری
- **تصاویر**: داده‌های پیکسلی باینری
- **قابل اجرا**: دستورالعمل‌های ماشین باینری

## تشبیه دنیای واقعی

باینری را مانند خانه سوئیچ نور تصور کنید:

| مفهوم | خانه سوئیچ نور | توضیح |
|--------|-----------------|--------|
| **بیت** | یک سوئیچ نور | روشن/خاموش |
| **بایت** | اتاق با ۸ سوئیچ | الگوی نوری نشان‌دهنده اطلاعات |
| **داده** | کل خانه | مجموعه‌ای از سوئیچ‌های هماهنگ |
| **کامپیوتر** | شبکه برق | منبع تغذیه |

## مزایای باینری

### قابلیت اعتماد
- **مدارهای ساده**: قطعات کمتر به معنای خرابی کمتر
- **تشخیص خطا**: تشخیص بیت‌های نادرست آسان است
- **افزونگی**: بیت‌های اضافی می‌توانند خطاها را تشخیص و تصحیح کنند

### سرعت
- **پردازش موازی**: چندین بیت همزمان پردازش می‌شوند
- **بهینه‌سازی سخت‌افزار**: مدارها مخصوص باینری طراحی شده‌اند
- **زمان‌بندی قابل پیش‌بینی**: سرعت عملیات ثابت

### جهانی بودن
- **هر اطلاعاتی**: می‌تواند متن، اعداد، تصاویر، صدا را نمایش دهد
- **استانداردسازی**: همان سیستم در سراسر جهان کار می‌کند
- **سازگاری**: همه کامپیوترها از پایه باینری یکسانی استفاده می‌کنند

## تصورات غلط رایج درباره باینری

### "باینری کند است"
**واقعیت**: عملیات باینری بسیار سریع هستند به لطف سخت‌افزار بهینه‌سازی شده.

### "باینری برای انسان‌ها سخت است"
**واقعیت**: ما از زبان‌های برنامه‌نویسی استفاده می‌کنیم که به طور خودکار به باینری تبدیل می‌شوند.

### "اعشار طبیعی‌تر است"
**واقعیت**: سیستم پایه-۱۰ ما فرهنگی است، نه اساسی برای محاسبات.

## نکات کلیدی

۱. **باینری با واقعیت الکتریکی سخت‌افزار همخوانی دارد**
۲. **دو حالت (۰/۱) امکان محاسبات قابل اعتماد و سریع را فراهم می‌کند**
۳. **همه داده‌ها در نهایت به باینری تبدیل می‌شوند**
۴. **عملیات باینری پایه همه محاسبات هستند**
۵. **لایه‌های انتزاعی پیچیدگی را پنهان می‌کنند در حالی که از قدرت باینری بهره می‌برند**

## سوالات رایج

### "آیا می‌توان مستقیماً کد ماشین نوشت؟"
- **پیچیده**: میلیون‌ها دستورالعمل باینری مورد نیاز است
- **خطاپر**: یک بیت اشتباه همه چیز را خراب می‌کند
- **غیرقابل مدیریت**: برای انسان غیرممکن است خواندن یا اشکال‌زدایی

### "چرا Python فایل‌های .pyc ایجاد می‌کند؟"
- **عملکرد**: بایت‌کد سریع‌تر از تفسیر مجدد منبع اجرا می‌شود
- **کش**: زمان را در اجرای بعدی ذخیره می‌کند
- **توزیع**: می‌توان بایت‌کد را بدون افشای منبع به اشتراک گذاشت

### "آیا می‌توان کد ماشین را به منبع بازگرداند؟"
- **دекомپایل**: ممکن اما ناقص
- **اطلاعات از دست رفته**: نام متغیرها، کامنت‌ها ناپدید می‌شوند
- **مسائل قانونی**: اغلب نقض مجوزهای نرم‌افزاری است

## مطالعه بیشتر
- محاسبات باینری را به تفصیل مطالعه کنید
- نمایش ممیز شناور را بیاموزید
- کدهای تصحیح خطا و فشرده‌سازی داده‌ها را بررسی کنید
- ماشین‌های مجازی را کاوش کنید
