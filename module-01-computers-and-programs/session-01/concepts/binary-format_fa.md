# فرمت باینری: زبان کامپیوترها

## به زبان ساده

**چه چیزی یاد می‌گیرید:** احتمالاً شنیده‌اید که کامپیوترها از «۰ و ۱» استفاده می‌کنند—اما چرا؟ و چگونه دو رقم می‌توانند همه چیز—متن، تصویر، موسیقی، ویدیو—را نمایش دهند؟ این مقاله دلیل الکتریکی ساده پشت باینری، نحوه مقیاس‌پذیری آن برای نمایش هر داده، و چرا نیازی به «فکر کردن به باینری» برای برنامه‌نویسی ندارید (زبان‌ها آن را برای شما انجام می‌دهند) توضیح می‌دهد.

**نکته برای تازه‌کاران:** نیازی به تبدیل اعداد به باینری برای نوشتن کد ندارید. زبان‌های برنامه‌نویسی این کار را به طور خودکار انجام می‌دهند. درک باینری به شما کمک می‌کند *چرا* کامپیوترها اینگونه کار می‌کنند—و چرا اندازه فایل‌ها با بایت، کیلوبایت و گیگابایت اندازه‌گیری می‌شود.

---

## چرا باینری؟ واقعیت الکتریکی

کامپیوترها ماشین‌های الکتریکی هستند. در پایین‌ترین سطح، آن‌ها فقط دو حالت را درک می‌کنند: **روشن** (جریان برق) و **خاموش** (بدون جریان برق). «نیمه روشن» یا «متوسط» وجود ندارد—فقط دو حالت واضح. به همین دلیل کامپیوترها از **باینری** (پایه-۲) استفاده می‌کنند: کاملاً با این واقعیت الکتریکی مطابقت دارد. ما این دو حالت را ۰ و ۱ می‌نامیم.

## باینری چیست؟

**باینری** یک سیستم عددی است که فقط دو رقم دارد: ۰ و ۱.

### مقایسه با دهدهی
| سیستم | پایه | رقم‌ها | مثال |
|--------|------|--------|---------|
| **دهدهی** | ۱۰ | ۰-۹ | ۴۲ |
| **باینری** | ۲ | ۰-۱ | ۱۰۱۰۱۰ |

### تبدیل بین سیستم‌ها
```
دهدهی ۵ = باینری ۱۰۱
دهدهی ۱۰ = باینری ۱۰۱۰
دهدهی ۱۵ = باینری ۱۱۱۱
```

## چرا کامپیوترها به باینری نیاز دارند؟

### ۱. **سادگی الکتریکی**
- **ترانزیستورها**: سوئیچ‌های الکترونیکی که روشن یا خاموش هستند
- **قابل اعتماد**: فقط دو حالت به معنای خطاهای کمتر است
- **سریع**: مدارهای ساده برای عملیات پایه

### ۲. **منطق دیجیتال**
باینری منطق بولی را فعال می‌کند:
- **AND**: هر دو ورودی باید ۱ باشند
- **OR**: حداقل یکی از ورودی‌ها باید ۱ باشد
- **NOT**: ۰ را به ۱ و ۱ را به ۰ تبدیل می‌کند

### ۳. **قابل توسعه**
- **بیت‌ها ترکیب می‌شوند**: ۸ بیت = ۱ بایت (۲۵۶ ترکیب)
- **رشد نمایی**: بیت‌های بیشتر = امکانات بیشتر
- **انعطاف‌پذیر**: می‌تواند هر نوع داده‌ای را نمایش دهد

## نمایش داده‌های باینری

### اعداد
```
دهدهی: ۴۲
باینری:  ۰۰۱۰۱۰۱۰
         ││││││││
         ۳۲+۸+۲=۴۲
```

### متن (ASCII)
```
'A' = ۰۱۰۰۰۰۰۱ (دهدهی ۶۵)
'B' = ۰۱۰۰۰۰۱۰ (دهدهی ۶۶)
'Hello' = ۰۱۰۰۱۰۰۰ ۰۱۱۰۰۱۰۱ ۰۱۱۰۱۱۰۰ ۰۱۱۰۱۱۰۰ ۰۱۱۰۱۱۱۱
```

### رنگ‌ها (RGB)
```
قرمز خالص: ۱۱۱۱۱۱۱۱ ۰۰۰۰۰۰۰۰ ۰۰۰۰۰۰۰۰
          (قرمز ۲۵۵، سبز ۰، آبی ۰)
```

### تصاویر و ویدیوها
- **پیکسل‌ها**: هر پیکسل مقادیر RGB باینری است
- **فشرده‌سازی**: الگوریتم‌ها اندازه باینری را کاهش می‌دهند
- **فرمت‌ها**: JPEG، PNG، MP4 همه به داده‌های باینری تبدیل می‌شوند

## عملیات باینری

### ریاضیات پایه
```
جمع:   ۵ + ۳
      ۰۱۰۱
    + ۰۰۱۱
    -------
      ۱۰۰۰ (۸)

تفریق: مشابه دهدهی با قرض گرفتن
ضرب: جمع مکرر
تقسیم: تفریق مکرر
```

### عملیات منطقی
```
AND: ۱ & ۱ = ۱، ۱ & ۰ = ۰، ۰ & ۰ = ۰
OR:  ۱ | ۱ = ۱، ۱ | ۰ = ۱، ۰ | ۰ = ۰
XOR: ۱ ^ ۱ = ۰، ۱ ^ ۰ = ۱، ۰ ^ ۰ = ۰
NOT: ~۱ = ۰، ~۰ = ۱
```

## واحدهای ذخیره‌سازی داده

### بیت (رقم باینری)
- کوچک‌ترین واحد: ۰ یا ۱
- یک حالت الکتریکی را نمایش می‌دهد

### بایت (۸ بیت)
- واحد استاندارد: ۲۵۶ مقدار ممکن (۰-۲۵۵)
- برای کاراکترها و اعداد کوچک استفاده می‌شود

### واحدهای بزرگ‌تر
```
۱ کیلوبایت (KB) = ۱,۰۰۰ بایت (در واقع ۱,۰۲۴)
۱ مگابایت (MB) = ۱,۰۰۰ KB
۱ گیگابایت (GB) = ۱,۰۰۰ MB
۱ ترابایت (TB) = ۱,۰۰۰ GB
```

## باینری در برنامه‌نویسی

### کد ماشین
برنامه‌ها به دستورالعمل‌های باینری تبدیل می‌شوند:
```
اسمبلی: ADD R1, R2, R3
باینری:   ۰۰۰۰۰۱ ۰۰۰۱۰ ۰۰۰۱۱ ۰۰۰۰۰ ۱۰۰۰۰۰
```

### انواع داده
انواع مختلف به فرمت‌های باینری مختلف نیاز دارند:
- **عدد صحیح**: نمایش مستقیم باینری
- **عدد اعشاری**: فرمت خاص (علامت + توان + مانتیس)
- **متن**: کدگذاری کاراکتر (ASCII، Unicode)

### فرمت‌های فایل
همه فایل‌ها در پایین‌ترین سطح باینری هستند:
- **فایل‌های متنی**: کاراکترهای کدگذاری شده باینری
- **تصاویر**: داده‌های پیکسلی باینری
- **فایل‌های اجرایی**: دستورالعمل‌های ماشین باینری

## تشبیه دنیای واقعی

باینری را مانند خانه‌ای با سوئیچ‌های چراغی تصور کنید:

| مفهوم | خانه سوئیچ چراغی |
|---------|-------------------|
| **بیت** | سوئیچ چراغی واحد (روشن/خاموش) |
| **بایت** | اتاق با ۸ سوئیچ |
| **داده** | الگوی چراغ‌ها که اطلاعات را نمایش می‌دهد |
| **کامپیوتر** | کل خانه با سوئیچ‌های هماهنگ |

## مزایای باینری

### قابلیت اعتماد
- **مدارهای ساده**: اجزای کمتر به معنای خرابی کمتر
- **تشخیص خطا**: تشخیص بیت‌های نادرست آسان است
- **افزونگی**: بیت‌های اضافی می‌توانند خطاها را تشخیص و تصحیح کنند

### سرعت
- **پردازش موازی**: چندین بیت همزمان پردازش می‌شوند
- **بهینه‌سازی سخت‌افزار**: مدارها مخصوصاً برای باینری طراحی شده‌اند
- **زمان‌بندی قابل پیش‌بینی**: سرعت‌های عملکرد ثابت

### جهانی بودن
- **هر اطلاعاتی**: می‌تواند متن، اعداد، تصاویر، صدا را نمایش دهد
- **استانداردسازی**: سیستم یکسان در سراسر جهان کار می‌کند
- **سازگاری**: همه کامپیوترها از پایه باینری یکسانی استفاده می‌کنند

## تصورات غلط رایج درباره باینری

### "باینری کند است"
**واقعیت**: عملیات باینری به دلیل سخت‌افزار بهینه‌سازی شده فوق‌العاده سریع هستند.

### "باینری برای انسان‌ها سخت است"
**واقعیت**: از زبان‌های برنامه‌نویسی استفاده می‌کنیم که به طور خودکار به باینری تبدیل می‌شوند.

### "دهدهی طبیعی‌تر است"
**واقعیت**: سیستم پایه-۱۰ ما فرهنگی است، نه بنیادی برای محاسبات.

## نکات کلیدی

۱. **باینری با واقعیت الکتریکی** سخت‌افزار کامپیوتر همخوانی دارد
۲. **دو حالت (۰/۱) امکان محاسبات قابل اعتماد و سریع را فراهم می‌کنند**
۳. **همه داده‌ها در پایین‌ترین سطح باینری می‌شوند**
۴. **عملیات باینری** پایه همه محاسبات هستند
۵. **تجریدها پیچیدگی را پنهان می‌کنند** در حالی که از قدرت باینری استفاده می‌کنند

## بررسی سریع (درک خود را بسنجید)

۱. چرا کامپیوترها نمی‌توانند به راحتی به جای باینری (۰–۱) از دهدهی (۰–۹) استفاده کنند؟
۲. ۸ بیت (۱ بایت) چند مقدار مختلف می‌تواند نمایش دهد؟
۳. وقتی حرف «A» را تایپ می‌کنید، در داخل کامپیوتر چه شکلی به خود می‌گیرد؟
