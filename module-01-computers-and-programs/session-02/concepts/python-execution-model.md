# Python's Execution Model: How Python Actually Runs Your Code

## In Plain Terms

**What you'll learn:** You write Python code in `.py` files, but how does it actually run? This article reveals Python's unique execution processâ€”the hybrid approach that combines compilation and interpretation, the role of the Python Virtual Machine (PVM), and how memory management works behind the scenes.

**Newbie tip:** Think of Python like a sophisticated restaurant with a prep kitchen and a cooking line. First, your recipe (source code) is prepped into standardized ingredients (bytecode). Then, the kitchen (PVM) cooks those ingredients. This two-step process is what makes Python both portable and reasonably fast.

## Quick Check

1. **Why does Python create .pyc files?**
   <details>
   <summary>Click for answer</summary>
   .pyc files store compiled bytecode. This avoids re-parsing and re-compiling source code every time you run a program, making subsequent runs faster. The bytecode is also platform-independent.
   </details>

2. **What does the PVM actually do?**
   <details>
   <summary>Click for answer</summary>
   The Python Virtual Machine is a stack-based interpreter that executes bytecode instructions one by one. It manages the evaluation stack, handles memory for objects, and coordinates execution flow.
   </details>

3. **How does Python manage memory automatically?**
   <details>
   <summary>Click for answer</summary>
   Python uses reference countingâ€”tracking how many variables reference each object. When an object's reference count drops to zero (no variables point to it), Python automatically deletes the object and frees the memory.
   </details>
