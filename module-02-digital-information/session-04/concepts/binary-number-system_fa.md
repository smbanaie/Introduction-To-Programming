# سیستم عددی باینری: چگونه کامپیوترها می‌شمرند

## چرا باینری؟

کامپیوترها ماشین‌های الکتریکی هستند که با دو حالت کار می‌کنند: روشن و خاموش، ولتاژ بالا و ولتاژ پایین. این طبیعت الکتریکی اساسی تعیین می‌کند چرا کامپیوترها از سیستم عددی باینری استفاده می‌کنند - سیستمی که فقط دو رقم دارد: ۰ و ۱.

## درک سیستم‌های عددی

### دهدهی (پایه ۱۰)
سیستم عددی روزمره ما:
- **رقم‌ها**: ۰، ۱، ۲، ۳، ۴، ۵، ۶، ۷، ۸، ۹
- **پایه**: ۱۰
- **مقادیر جایگاه**: ۱۰^۰، ۱۰^۱، ۱۰^۲، ۱۰^۳...
- **مثال**: ۴۲ = ۴×۱۰^۱ + ۲×۱۰^۰

### باینری (پایه ۲)
سیستم عددی کامپیوتر:
- **رقم‌ها**: ۰، ۱
- **پایه**: ۲
- **مقادیر جایگاه**: ۲^۰، ۲^۱، ۲^۲، ۲^۳...
- **مثال**: ۱۰۱۰₂ = ۱×۲^۳ + ۰×۲^۲ + ۱×۲^۱ + ۰×۲^۰ = ۸ + ۲ = ۱۰

## مقادیر جایگاه باینری

### توان‌های ۲
هر جایگاه توانی از ۲ را نمایش می‌دهد:
```
۲^۰ = ۱
۲^۱ = ۲
۲^۲ = ۴
۲^۳ = ۸
۲^۴ = ۱۶
۲^۵ = ۳۲
۲^۶ = ۶۴
۲^۷ = ۱۲۸
۲^۸ = ۲۵۶
```

### خواندن اعداد باینری
```
باینری:  ۱  ۰  ۱  ۰  ۱  ۰  ۱  ۰
جایگاه‌ها: ۱۲۸ ۶۴ ۳۲ ۱۶ ۸  ۴  ۲  ۱
مقدار:  ۱۲۸ + ۰ + ۳۲ + ۰ + ۸ + ۰ + ۲ + ۰ = ۱۷۰
```

## تبدیل بین دهدهی و باینری

### دهدهی به باینری
**روش**: تقسیم مکرر بر ۲

مثال: تبدیل ۴۲ به باینری
```
۴۲ ÷ ۲ = ۲۱ باقی‌مانده ۰
۲۱ ÷ ۲ = ۱۰ باقی‌مانده ۱
۱۰ ÷ ۲ = ۵  باقی‌مانده ۰
۵ ÷ ۲ = ۲   باقی‌مانده ۱
۲ ÷ ۲ = ۱   باقی‌مانده ۰
۱ ÷ ۲ = ۰   باقی‌مانده ۱

باقی‌مانده‌ها را پایین به بالا بخوانید: ۱۰۱۰۱۰₂
```

### باینری به دهدهی
**روش**: هر بیت را در مقدار جایگاه آن ضرب کنید

مثال: تبدیل ۱۰۱۰۱۰₂ به دهدهی
```
۱×۳۲ + ۰×۱۶ + ۱×۸ + ۰×۴ + ۱×۲ + ۰×۱ = ۳۲ + ۸ + ۲ = ۴۲
```

## حساب باینری

### جمع
قوانین:
- ۰ + ۰ = ۰
- ۰ + ۱ = ۱
- ۱ + ۰ = ۱
- ۱ + ۱ = ۰ (حمل ۱)

مثال:
```
  ۱ ۱ ۱  (حمل‌ها)
    ۱ ۰ ۱  (۵)
  + ۰ ۱ ۱  (۳)
  -------
  ۱ ۰ ۰ ۰  (۸)
```

### تفریق
قوانین:
- ۰ - ۰ = ۰
- ۱ - ۰ = ۱
- ۱ - ۱ = ۰
- ۰ - ۱ = ۱ (وام ۱)

مثال:
```
  ۱ ۰  (وام‌ها)
    ۱ ۰ ۱  (۵)
  - ۰ ۰ ۱  (۱)
  -------
    ۰ ۱ ۰  (۴)
```

### ضرب
همان دهدهی، اما فقط ۰ها و ۱ها:
```
    ۱ ۰ ۱  (۵)
  × ۰ ۱ ۱  (۳)
  -------
    ۱ ۰ ۱
  ۱ ۰ ۱
----------
  ۱ ۱ ۱ ۱  (۱۵)
```

## باینری در معماری کامپیوتر

### اندازه داده‌ها
```
۱ بیت = کوچک‌ترین واحد
۱ بایت = ۸ بیت = ۲۵۶ مقدار ممکن
۱ کلمه = ۳۲ یا ۶۴ بیت (بسته به پردازنده)
```

### آدرس‌دهی حافظه
هر موقعیت حافظه آدرس باینری خود را دارد:
```
آدرس ۰: ۰۰۰۰۰۰۰۰
آدرس ۱: ۰۰۰۰۰۰۰۱
آدرس ۲۵۵: ۱۱۱۱۱۱۱۱
```

### انواع داده
انواع مختلف به الگوهای بیت مختلف نیاز دارند:
```
بولی: ۱ بیت (درست/غلط)
بایت: ۸ بیت (۰-۲۵۵)
عدد صحیح: ۳۲ بیت (-۲ میلیارد تا +۲ میلیارد)
عدد اعشاری: ۳۲ بیت (اعداد اعشاری)
```

## هگزادسیمال: دوست باینری

### چرا هگزادسیمال؟
اعداد باینری خیلی زود طولانی می‌شوند. هگزادسیمال (پایه ۱۶) نمایش فشرده‌تری فراهم می‌کند:
- **رقم‌ها**: ۰-۹، A-F (A=۱۰، B=۱۱، C=۱۲، D=۱۳، E=۱۴، F=۱۵)
- **تبدیل**: ۴ بیت باینری = ۱ رقم هگز

### تبدیل باینری-هگز
```
باینری:  ۱۱۱۱ ۱۰۱۰ ۱۱۰۰ ۱۱۱۱
هگز:       F    A    C    F
```

### آدرس‌های حافظه
هگز معمولاً برای آدرس‌های حافظه استفاده می‌شود:
```
آدرس 0x1000 = ۰۰۰۱۰۰۰۰۰۰۰۰۰۰۰۰ در باینری
آدرس 0xFFFF = ۱۱۱۱۱۱۱۱۱۱۱۱۱۱۱ در باینری
```

## اعداد باینری علامت‌دار

### علامت-قدر مطلق
بیت سمت چپ علامت را نمایش می‌دهد:
- ۰ = مثبت، ۱ = منفی
- مثال: +۵ = ۰۱۰۱، -۵ = ۱۱۰۱

### مکمل دو (استاندارد مدرن)
- اعداد مثبت: باینری عادی
- اعداد منفی: همه بیت‌ها را برعکس کن و ۱ اضافه کن
- مثال: -۵ = ۱۱۱۱۱۰۱۱ (۰۱۰۱ را به ۱۰۱۰ برعکس کن، ۱ اضافه کن)

## اعداد اعشاری باینری

### استاندارد IEEE ۷۵۴
فرمت float ۳۲ بیتی:
- **علامت**: ۱ بیت
- **توان**: ۸ بیت (با بایاس ۱۲۷)
- **مانتیس**: ۲۳ بیت

### مثال: ۳.۱۴
```
علامت: ۰ (مثبت)
توان: ۱۰۰۰۰۰۰۰ (۱۲۸ - ۱۲۷ = ۱)
مانتیس: ۱۰۰۱۰۰۰۱۱۱۱۰۱۰۱۱۱۰۰۰۰۱۱
نتیجه: ۱.۵۷۰۷۹۶... × ۲^۱ = ۳.۱۴۱۵۹۲...
```

## الگوهای رایج باینری

### توان‌های ۲
```
۲^۰ = ۱        = ۰۰۰۱
۲^۱ = ۲        = ۰۰۱۰
۲^۲ = ۴        = ۰۱۰۰
۲^۳ = ۸        = ۱۰۰۰
۲^۴ = ۱۶       = ۰۰۰۱۰۰۰۰
```

### ماسک‌های بیت
```
بررسی بیت ۳: ۰۰۰۰۱۰۰۰ (۸)
تنظیم بیت ۲:   ۰۰۰۰۰۱۰۰ (۴)
پاک کردن بیت ۱: ۱۱۱۱۱۱۰۱ (۲۵۳)
```

## نکات کلیدی

۱. **باینری با سخت‌افزار کامپیوتر همخوانی دارد**: حالات الکتریکی روشن/خاموش
۲. **مقادیر جایگاه بزرگی را تعیین می‌کنند**: هر جایگاه توانی از ۲ است
۳. **تبدیل نیاز به تمرین دارد**: تبدیل دهدهی ↔ باینری
۴. **حساب به همان صورت کار می‌کند**: فقط با قوانین مختلف
۵. **هگز باینری را قابل مدیریت می‌کند**: نمایش فشرده برای انسان‌ها

## اشتباهات رایج

### صفرهای پیشین
- **غلط**: ۰۰۱۰₂ = ۲₁₀
- **درست**: ۰۰۱۰₂ = ۲₁₀ (صفرهای پیشین مقدار را تغییر نمی‌دهند)

### ترتیب بیت
- **غلط**: خواندن راست به چپ
- **درست**: بیت سمت چپ مقدار جایگاه بالاتری دارد

### سرریز
- **مشکل**: جمع ۸ بیتی ۲۵۵ + ۱ = ۰ (با حمل)
- **راه حل**: از انواع داده بزرگ‌تر برای جلوگیری از سرریز استفاده کنید

## مطالعه بیشتر
- حساب باینری را به تفصیل مطالعه کنید
- نمایش‌های مختلف عدد صحیح را بیاموزید (علامت‌دار/بدون علامت)
- حساب اعشاری و مسائل دقت را بررسی کنید
- سیستم‌های کدگذاری دهدهی باینری (BCD) را درک کنید
