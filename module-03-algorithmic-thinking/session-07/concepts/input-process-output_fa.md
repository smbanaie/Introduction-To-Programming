# ورودی-پردازش-خروجی: پایه حل مسئله

## IPO چیست؟

ورودی-پردازش-خروجی (IPO) یک مدل اساسی برای درک نحوه کار سیستم‌ها است. هر برنامه، هر الگوریتم و هر سیستم کامپیوتری از این الگوی پایه پیروی می‌کند: ورودی دریافت کن، آن را پردازش کن و خروجی تولید کن.

## مدل IPO

```
ورودی → پردازش → خروجی

ورودی: داده یا اطلاعات دریافت شده از منابع خارجی
پردازش: عملیات روی داده ورودی انجام شده
خروجی: نتایج یا اطلاعات تولید شده توسط پردازش
```

## ورودی: دریافت داده‌ها

### انواع ورودی
- **ورودی کاربر**: داده‌های ارائه شده توسط افراد (کیبورد، ماوس، لمس)
- **ورودی فایل**: داده‌های خوانده شده از دستگاه‌های ذخیره‌سازی
- **ورودی سنسور**: داده‌ها از سنسورهای فیزیکی (دما، نور، حرکت)
- **ورودی شبکه**: داده‌های دریافت شده از شبکه‌ها (اینترنت، APIها)

### ویژگی‌های ورودی
- **فرمت**: نحوه ساختار داده‌ها (متن، اعداد، باینری)
- **اعتبارسنجی**: اطمینان از برآورده کردن الزامات ورودی
- **مدیریت خطا**: چه کاری با ورودی‌های نامعتبر یا گم‌شده انجام دهیم
- **زمان‌بندی**: چه زمانی ورودی دریافت می‌شود (فوری، برنامه‌ریزی شده، رویدادمحور)

### مثال‌های ورودی
```python
# ورودی کیبورد
name = input("نام خود را وارد کنید: ")

# ورودی فایل
with open("data.txt", "r") as file:
    content = file.read()

# ورودی سنسور (شبیه‌سازی شده)
temperature = read_temperature_sensor()
```

## پردازش: تبدیل داده‌ها

### مؤلفه‌های پردازش
- **منطق**: تصمیم‌گیری بر اساس ورودی
- **محاسبات**: عملیات ریاضی
- **دستکاری داده**: مرتب‌سازی، فیلتر کردن، تبدیل
- **عملیات ذخیره‌سازی**: ذخیره نتایج میانی

### ویژگی‌های پردازش
- **ترتیبی**: مراحل به ترتیب اجرا می‌شوند
- **مشروط**: مسیرهای مختلف بر اساس شرایط
- **تکراری**: عملیات روی داده‌ها تکرار می‌شوند
- **وضعیتی**: نتایج عملیات قبلی را به یاد می‌آورد

### مثال‌های پردازش
```python
# پردازش محاسبه
def calculate_average(scores):
    total = sum(scores)        # مجموع همه امتیازها
    count = len(scores)        # شمارش آیتم‌ها
    average = total / count    # محاسبه میانگین
    return average             # بازگشت نتیجه

# پردازش تبدیل متن
def process_text(text):
    text = text.lower()        # تبدیل به حروف کوچک
    text = text.strip()        # حذف فاصله‌های اضافی
    words = text.split()       # تقسیم به کلمات
    return words               # بازگشت لیست کلمات
```

## خروجی: تولید نتایج

### انواع خروجی
- **خروجی نمایش**: اطلاعات روی صفحه نمایش داده می‌شود
- **خروجی فایل**: داده‌ها در ذخیره‌سازی ذخیره می‌شوند
- **خروجی چاپ**: اسناد فیزیکی یا برچسب‌ها
- **خروجی شبکه**: داده‌ها روی شبکه‌ها ارسال می‌شوند
- **خروجی عملگر**: اقدامات فیزیکی (موتورها، چراغ‌ها، صداها)

### ویژگی‌های خروجی
- **فرمت**: نحوه ارائه نتایج (متن، گرافیک، صوتی)
- **مقصد**: کجا خروجی می‌رود (صفحه، فایل، شبکه)
- **پایداری**: چقدر خروجی باقی می‌ماند (موقتی، دائمی)
- **بازخورد**: تأیید تحویل موفق خروجی

### مثال‌های خروجی
```python
# خروجی صفحه
print(f"سلام، {name}!")

# خروجی فایل
with open("results.txt", "w") as file:
    file.write("پردازش کامل شد\n")

# خروجی گرافیکی (شبیه‌سازی شده)
draw_circle(x=100, y=100, radius=50, color="blue")
```

## IPO در سیستم‌های دنیای واقعی

### دستگاه قهوه
```
ورودی: دانه‌های قهوه، آب، فشار دکمه روشن
پردازش:
  - آسیاب دانه‌ها → گرم کردن آب → دم کردن قهوه → مخلوط کردن
خروجی: قهوه داغ در فنجان
```

### سیستم ایمیل
```
ورودی: پیام ایمیل (به، موضوع، بدنه، پیوست‌ها)
پردازش:
  - اعتبارسنجی آدرس‌ها → قالب‌بندی پیام → اتصال به سرور → تحویل به گیرندگان
خروجی: تأیید ارسال، رسیدهای تحویل، اعلان‌های خطا
```

### موتور جستجوی وب
```
ورودی: عبارت جستجو + ترجیحات کاربر
پردازش:
  - پارس عبارت → جستجو در شاخص → رتبه‌بندی نتایج → فیلتر کردن هرزنامه
خروجی: صفحه نتایج جستجو
```

## IPO در برنامه‌نویسی

### ساختار تابع
هر تابع از IPO پیروی می‌کند:
```python
def process_data(data):
    # ورودی: پارامتر data
    # پردازش: اعتبارسنجی و تبدیل
    if not data:
        return None  # خروجی زودهنگام

    processed = data.upper().strip()
    word_count = len(processed.split())

    # خروجی: بازگشت نتایج
    return {
        "text": processed,
        "words": word_count
    }
```

### ساختار برنامه
برنامه‌های کامل سیستم‌های IPO هستند:
```python
def main():
    # مرحله ورودی
    filename = input("نام فایل را وارد کنید: ")
    operation = input("عملیات را انتخاب کنید (count/read): ")

    # مرحله پردازش
    try:
        with open(filename, "r") as file:
            content = file.read()

        if operation == "count":
            result = len(content.split())
            result_text = f"فایل شامل {result} کلمه است"
        elif operation == "read":
            result_text = content[:100] + "..."
        else:
            result_text = "عملیات نامعتبر"

    except FileNotFoundError:
        result_text = "فایل یافت نشد"

    # مرحله خروجی
    print(result_text)

if __name__ == "__main__":
    main()
```

## IPO و وضعیت

### پردازش وضعیتی
الگوریتم‌هایی که اطلاعات را بین عملیات به یاد می‌آورند:

```python
def running_average():
    total = 0
    count = 0

    def add_number(number):
        nonlocal total, count  # دسترسی به متغیرهای بیرونی (وضعیت)
        total += number
        count += 1
        return total / count

    return add_number

# استفاده
avg_func = running_average()
print(avg_func(10))  # ۱۰.۰ (وضعیت: total=10, count=1)
print(avg_func(20))  # ۱۵.۰ (وضعیت: total=30, count=2)
print(avg_func(30))  # ۲۰.۰ (وضعیت: total=60, count=3)
```

### پردازش بدون وضعیت
هر عملیات مستقل است، هیچ حافظه‌ای از عملیات قبلی ندارد:

```python
def calculate_average(numbers):
    # بدون وضعیت - هر فراخوانی مستقل است
    if not numbers:
        return 0
    return sum(numbers) / len(numbers)

# هر فراخوانی تازه شروع می‌شود
print(calculate_average([10, 20, 30]))  # ۲۰.۰
print(calculate_average([40, 50]))      # ۴۵.۰
print(calculate_average([10, 20, 30]))  # ۲۰.۰ (نتیجه یکسان)
```

## IPO در طراحی الگوریتم

### حل مسئله گام به گام
۱. **ورودی را تعریف کن**: چه اطلاعاتی در دسترس است؟
۲. **خروجی را مشخص کن**: نتیجه نهایی چگونه باید باشد؟
۳. **پردازش را طراحی کن**: چه مرحله‌هایی ورودی را به خروجی تبدیل می‌کنند؟
۴. **موارد لبه را مدیریت کن**: اگر ورودی نامعتبر یا غیرمعمول باشد چه؟
۵. **نتایج را تأیید کن**: چگونه مطمئن شویم خروجی صحیح است؟

### مثال: سیستم نمره‌دهی
```
ورودی: مقدار نمره + واحد (سلسیوس/فارنهایت)
پردازش:
  - اگر ورودی سلسیوس باشد: تبدیل به فارنهایت
  - اگر ورودی فارنهایت باشد: تبدیل به سلسیوس
  - فرمول: F = C × 9/5 + 32, C = (F - 32) × 5/9
خروجی: دمای تبدیل شده + واحد جدید
```

## الگوهای رایج IPO

### الگوی اعتبارسنجی داده
```
ورودی: داده‌های خام
پردازش: بررسی فرمت، محدوده، کامل بودن
خروجی: داده‌های معتبر یا پیام خطا
```

### الگوی تبدیل داده
```
ورودی: داده در فرمت A
پردازش: ترجمه/تبدیل به فرمت B
خروجی: داده در فرمت B
```

### الگوی تصمیم‌گیری
```
ورودی: داده‌های وضعیت + معیارها
پردازش: ارزیابی در برابر قوانین
خروجی: تصمیم + استدلال
```

### الگوی تجمیع
```
ورودی: مجموعه‌ای از آیتم‌ها
پردازش: هر آیتم را پردازش کن، نتایج را جمع کن
خروجی: نتیجه خلاصه/ترکیبی
```

## IPO و مدیریت خطا

### اعتبارسنجی ورودی
```python
def safe_divide(a, b):
    # اعتبارسنجی ورودی
    if not isinstance(a, (int, float)) or not isinstance(b, (int, float)):
        return "خطا: انواع ورودی نامعتبر"

    # پردازش
    if b == 0:
        return "خطا: تقسیم بر صفر"

    result = a / b

    # خروجی
    return result
```

### بازیابی خطا
```python
def robust_file_reader(filename):
    try:
        # ورودی
        with open(filename, "r") as file:
            content = file.read()

        # پردازش
        lines = content.split('\n')
        word_count = sum(len(line.split()) for line in lines)

        # خروجی
        return f"فایل شامل {len(lines)} خط و {word_count} کلمه است"

    except FileNotFoundError:
        return "خطا: فایل یافت نشد"
    except PermissionError:
        return "خطا: مجوز خواندن فایل وجود ندارد"
    except Exception as e:
        return f"خطا: {str(e)}"
```

## نکات کلیدی

۱. **IPO جهانی است**: هر سیستم از ورودی-پردازش-خروجی پیروی می‌کند
۲. **جداسازی واضح**: هر مرحله مسئولیت‌های متمایزی دارد
۳. **مدیریت خطا**: ورودی‌ها را به زیبایی اعتبارسنجی کن، خطاهای پردازش را مدیریت کن، تحویل خروجی را تأیید کن
۴. **وضعیت مهم است**: برخی فرآیندها اطلاعات را بین عملیات به یاد می‌آورند
۵. **طراحی مدولار**: سیستم‌های پیچیده را به مؤلفه‌های IPO تقسیم کن
