# جلسه ۱۰: شروع کار با پایتون – محیط و اولین گام‌ها

## مرور جلسه

به اولین جلسه عملی برنامه‌نویسی خود خوش آمدید! بعد از نه هفته ساخت پایه‌های مفهومی، امروز بالاخره شروع به نوشتن و اجرای کد پایتون خواهیم کرد. یک محیط پایتون راه‌اندازی خواهیم کرد، REPL تعاملی را بررسی خواهیم کرد، اولین اسکریپت خود را ایجاد خواهیم کرد، و آنچه درباره الگوریتم‌ها و نمایش داده‌ها یاد گرفته‌ایم را به نحو واقعی پایتون پیوند خواهیم داد. این نشان‌دهنده انتقال از فکر کردن درباره برنامه‌نویسی به انجام برنامه‌نویسی است.

## اصطلاحات کلیدی

- **مفسر پایتون**: برنامه‌ای که کد پایتون را می‌خواند و اجرا می‌کند
- **REPL (حلقه خواندن-ارزیابی-چاپ)**: محیط پایتون تعاملی برای اجرای فوری کد
- **اسکریپت**: فایلی که شامل کد پایتون با پسوند .py است
- **پایتون شل/IDLE**: محیط توسعه داخلی که با پایتون همراه است
- **خط فرمان/ترمینال**: رابط مبتنی بر متن برای اجرای برنامه‌ها

## مفسر پایتون چیست؟

مفسر پایتون برنامه‌ای است که:
- کد منبع پایتون شما را می‌خواند
- آن را به بایت‌کد (فرم میانی) ترجمه می‌کند
- بایت‌کد را با استفاده از ماشین مجازی پایتون (PVM) اجرا می‌کند
- مدیریت حافظه، I/O و سایر جزئیات سطح پایین را مدیریت می‌کند

این چیزی است که کد قابل خواندن توسط انسان شما را روی یک کامپیوتر قابل اجرا می‌کند.

## راه‌اندازی پایتون

### گزینه ۱: نصب پایتون به صورت محلی
۱. از python.org دیدن کنید و آخرین نسخه را دانلود کنید
۲. نصب‌کننده را اجرا کنید (تیک "Add Python to PATH" را بزنید)
۳. خط فرمان را باز کنید و تایپ کنید `python --version`

### گزینه ۲: استفاده از محیط‌های آنلاین
- Replit.com (محیط کد نویسی آنلاین رایگان)
- PythonAnywhere.com
- Google Colab (برای علوم داده، اما برای مبانی کار می‌کند)

### گزینه ۳: استفاده از IDLE (با پایتون همراه است)
- در منوی Start خود به دنبال "IDLE" بگردید
- ویرایشگر و مفسر داخلی

## استفاده از REPL پایتون

REPL برای آزمایش عالی است:

۱. خط فرمان را باز کنید
۲. تایپ کنید `python` و Enter را بزنید
۳. پرامپت `>>>` را خواهید دید
۴. عبارت‌ها را تایپ کنید و نتایج فوری ببینید:

```python
>>> 2 + 3
5
>>> "Hello" + " World"
'Hello World'
>>> print("Welcome to Python!")
Welcome to Python!
```

برای خروج: `Ctrl+D` (Linux/Mac) یا `Ctrl+Z` سپس Enter (Windows)

## اولین اسکریپت پایتون شما

بیایید یک برنامه ساده ایجاد کنیم:

۱. **یک فایل** به نام `hello.py` ایجاد کنید
۲. **این کد را اضافه کنید**:
```python
print("Hello, World!")
name = input("What's your name? ")
print("Nice to meet you, " + name + "!")
```

۳. **آن را اجرا کنید**:
   - خط فرمان: `python hello.py`
   - IDLE: فایل را باز کنید و F5 را بزنید
   - آنلاین: از دکمه run استفاده کنید

## ارتباط با شبه‌کد

شبه‌کد ما را از جلسه ۹ به یاد دارید؟ در اینجا نحوه ترجمه آن آمده است:

**شبه‌کد:**
```
DISPLAY "Hello, World!"
GET name FROM user
DISPLAY "Nice to meet you, " + name + "!"
```

**پایتون:**
```python
print("Hello, World!")
name = input("What's your name? ")
print("Nice to meet you, " + name + "!")
```

مفاهیم یکسان هستند - نحو فقط روش پایتون برای بیان آنهاست.

## عناصر نحو پایه پایتون

### کامنت‌ها
```python
# این یک کامنت است - توسط پایتون نادیده گرفته می‌شود
print("Hello")  # کامنت‌ها می‌توانند بعد از کد هم باشند
```

### دستورات
هر خط معمولاً یک دستور است:
```python
print("First")
print("Second")
```

### عبارت‌ها
کدی که به یک مقدار ارزیابی می‌شود:
```python
2 + 3          # عبارت حسابی
"Hello" + "!"  # اتصال رشته
len("Python")  # فراخوانی تابع
```

## مسائل رایج مبتدیان

### مسائل ذخیره فایل
- مطمئن شوید فایل با `.py` تمام شود
- در مکانی که بتوانید پیدا کنید ذخیره کنید
- کدگذاری فایل را چک کنید (معمولاً UTF-8)

### مسائل مسیر
- از `cd` برای ناوبری به دایرکتوری اسکریپت خود استفاده کنید
- یا از مسیرهای کامل استفاده کنید: `python C:\Users\YourName\hello.py`

### پایتون یافت نشد
- چک کنید آیا پایتون نصب است: `python --version`
- به جای `python` از `python3` امتحان کنید
- در صورت نیاز مجدداً نصب کنید

### حساسیت به حالت
پایتون به حالت حساس است:
- `Print("hello")` ❌ (P بزرگ)
- `print("hello")` ✅
