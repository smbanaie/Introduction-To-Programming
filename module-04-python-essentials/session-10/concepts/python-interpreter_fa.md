# مفسر پایتون: مترجم کدهای شما

## چه چیزهایی یاد خواهید گرفت
- مفسر پایتون چه کاری انجام می‌دهد
- چگونه کد پایتون را به دو روش مختلف اجرا کنیم
- چرا پایتون "زبان تفسیری" نامیده می‌شود

---

## مفسر پایتون چیست؟

مفسر پایتون را مانند یک **مترجم** در نظر بگیرید که بین شما و کامپیوتر قرار دارد.

```
شما (کد پایتون می‌نویسید)
    ↓
مفسر پایتون (ترجمه می‌کند)
    ↓
کامپیوتر (اجرا می‌کند)
```

مفسر کد پایتون شما را خط به خط می‌خواند و به کامپیوتر می‌گوید چه کاری انجام دهد. برخلاف برخی زبان‌های برنامه‌نویسی دیگر که نیاز به "کامپایل" (ترجمه همه‌چیز یک‌باره قبل از اجرا) دارند، پایتون کد شما را خط به خط ترجمه و اجرا می‌کند.

### تشبیه: مفسر مثل آشپزی است که دستورالعمل می‌خواند

فرض کنید دارید با یک آشپز آشپزی می‌کنید:
- **زبان‌های کامپایل شده**: کل دستور غذا را می‌نویسید، آشپز آن را حفظ می‌کند، بعد همه‌چیز را می‌پزد
- **پایتون (تفسیری)**: هر مرحله را بلند می‌خوانید و آشپز فوراً آن را انجام می‌دهد

این ویژگی پایتون را برای یادگیری عالی می‌کند چون بازخورد فوری می‌گیرید!

---

## دو روش برای اجرای کد پایتون

### روش ۱: حالت تعاملی (REPL)

REPL مخفف **R**ead (خواندن)، **E**valuate (ارزیابی)، **P**rint (چاپ)، **L**oop (حلقه) است. مثل یک مکالمه با پایتون است.

#### چگونه حالت تعاملی را شروع کنیم

ترمینال/خط فرمان خود را باز کنید و تایپ کنید:

```bash
python
```

یا در برخی سیستم‌ها (مخصوصاً macOS/Linux):

```bash
python3
```

چیزی شبیه به این را خواهید دید:

```
Python 3.9.7 (default, Sep 16 2021, 13:09:58)
[GCC 7.5.0] :: Anaconda, Inc. on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>
```

`>>>` را **prompt** (درخواست‌کننده) می‌نامند - یعنی پایتون می‌گوید "آماده‌ام، چیزی تایپ کن!"

#### خودتان امتحان کنید!

این خطوط را یکی‌یکی بعد از `>>>` تایپ کنید:

```python
>>> print("سلام، دنیا!")
سلام، دنیا!

>>> 5 + 3
8

>>> name = "علی"
>>> print(name)
علی
```

توجه کنید که پایتون فوراً بعد از هر خط نتیجه را به شما نشان می‌دهد!

#### چه زمانی از حالت تعاملی استفاده کنیم

✅ **عالی برای:**
- تست کردن قطعه کدهای کوچک
- یادگیری و آزمایش
- محاسبات سریع
- بررسی اینکه آیا چیزی کار می‌کند یا نه

❌ **مناسب نیست برای:**
- برنامه‌هایی که می‌خواهید ذخیره کنید
- پروژه‌های بزرگ
- کدی که می‌خواهید به اشتراک بگذارید

---

### روش ۲: حالت اسکریپت (نوشتن در فایل)

حالت اسکریپت یعنی کد پایتون خود را در یک فایل بنویسید و همه‌اش را یک‌باره اجرا کنید.

#### مرحله ۱: ایجاد یک فایل پایتون

یک فایل به نام `hello.py` با این محتوا ایجاد کنید:

```python
# اولین برنامه پایتون من!
print("سلام، دنیا!")
name = input("اسمت چیه؟ ")
print(f"خوشحالم که آشنا شدم، {name}!")
```

**مهم:** فایل‌های پایتون همیشه با `.py` تمام می‌شوند

#### مرحله ۲: اجرای فایل

ترمینال را در همان پوشه فایل خود باز کنید و تایپ کنید:

```bash
python hello.py
```

یا در برخی سیستم‌ها:

```bash
python3 hello.py
```

خروجی:

```
سلام، دنیا!
اسمت چیه؟ علی
خوشحالم که آشنا شدم، علی!
```

#### چه زمانی از حالت اسکریپت استفاده کنیم

✅ **عالی برای:**
- برنامه‌هایی که می‌خواهید ذخیره و دوباره اجرا کنید
- اشتراک‌گذاری کد با دیگران
- ساخت برنامه‌های کامل
- نوشتن برنامه‌های طولانی‌تر

---

## وقتی پایتون را اجرا می‌کنید چه اتفاقی می‌افتد؟

نسخه ساده آنچه پشت صحنه اتفاق می‌افتد:

```
فایل کد شما (.py)
    ↓
پایتون فایل را می‌خواند
    ↓
پایتون به بایت‌کد تبدیل می‌کند (فرمت دوست‌دار کامپیوتر)
    ↓
ماشین مجازی پایتون (PVM) اجرا می‌کند
    ↓
نتیجه را می‌بینید!
```

نگران جزئیات فنی نباشید - پایتون همه این کارها را خودش انجام می‌دهد!

---

## نسخه‌های مختلف پایتون

ممکن است درباره پایتون ۲ و پایتون ۳ شنیده باشید. آنچه باید بدانید:

| نسخه | وضعیت | کدام را استفاده کنیم |
|------|-------|----------------------|
| پایتون ۲ | قدیمی، دیگر پشتیبانی نمی‌شود | ❌ استفاده نکنید |
| پایتون ۳ | فعلی، در حال توسعه | ✅ از این استفاده کنید! |

**همیشه از پایتون ۳** برای هر پروژه یا یادگیری جدید استفاده کنید.

### بررسی نسخه پایتون شما

در حالت تعاملی، نسخه در بالا نشان داده می‌شود. یا می‌توانید اجرا کنید:

```bash
python --version
# یا
python3 --version
```

## اشتباهات رایج مبتدی‌ها

### اشتباه ۱: فراموش کردن ذخیره قبل از اجرا

اگر تغییراتی در فایل `.py` خود ایجاد می‌کنید، باید قبل از اجرای دوباره **ذخیره** کنید. پایتون آنچه روی دیسک ذخیره شده را اجرا می‌کند، نه آنچه در ویرایشگر شماست.

### اشتباه ۲: پسوند فایل اشتباه

مطمئن شوید فایل‌های شما با `.py` تمام می‌شوند:
- ✅ `myprogram.py` (درست)
- ❌ `myprogram.txt` (اشتباه)
- ❌ `myprogram` (اشتباه - بدون پسوند)

### اشتباه ۳: قرار دادن `>>>` در فایل

وقتی یک فایل `.py` می‌نویسید، `>>>` را قرار ندهید:

```python
# ❌ اشتباه - این کار را نکنید
>>> print("سلام")

# ✅ درست - این کار را بکنید
print("سلام")
```

### اشتباه ۴: پوشه اشتباه

مطمئن شوید وقتی برنامه را اجرا می‌کنید در همان پوشه فایل پایتون خود هستید:

```bash
# ببینید در چه پوشه‌ای هستید
pwd  # در macOS/Linux
cd   # در ویندوز

# فایل‌ها را لیست کنید تا ببینید فایل پایتون شما آنجاست
ls        # در macOS/Linux
dir       # در ویندوز
```

## خودتان امتحان کنید: تمرین‌ها

### تمرین ۱: تمرین حالت تعاملی
1. با تایپ `python` یا `python3` حالت تعاملی را باز کنید
2. این محاسبات را امتحان کنید:
   - `10 * 5`
   - `100 / 4`
   - `2 ** 8` (یعنی ۲ به توان ۸)
3. یک متغیر بسازید: `favorite_number = 7`
4. آن را چاپ کنید: `print(favorite_number)`

### تمرین ۲: اولین اسکریپت شما
1. یک فایل به نام `first_program.py` بسازید
2. این کد را بنویسید:
   ```python
   print("اولین برنامه پایتون من")
   print("من دارم پایتون یاد می‌گیرم!")
   ```
3. فایل را ذخیره کنید
4. از ترمینال اجرا کنید: `python first_program.py`

### تمرین ۳: اسکریپت سلام تعاملی
1. یک فایل به نام `greeting.py` بسازید
2. این کد را بنویسید:
   ```python
   name = input("اسمت را وارد کن: ")
   age = input("سنت را وارد کن: ")
   print(f"سلام {name}! تو {age} سال داری.")
   ```
3. ذخیره و اجرا کنید

## نکات کلیدی

1. **مفسر** مثل یک مترجم بین شما و کامپیوتر است
2. **حالت تعاملی** برای تست‌های سریع و یادگیری عالی است (از دستور `python` استفاده کنید)
3. **حالت اسکریپت** برای نوشتن برنامه‌های کامل که می‌توانید ذخیره و به اشتراک بگذارید (فایل‌های `.py`)
4. **همیشه از پایتون ۳** استفاده کنید - پایتون ۲ قدیمی است
5. **فراموش نکنید ذخیره** کنید فایل‌هایتان را قبل از اجرا!
