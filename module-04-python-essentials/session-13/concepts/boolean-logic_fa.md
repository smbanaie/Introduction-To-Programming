# منطق بولی: درست یا نادرست؟

## چه چیزهایی یاد خواهید گرفت
- معنی True و False در برنامه‌نویسی
- نحوه استفاده از AND، OR و NOT
- جداول درستی برای عملیات منطقی
- وقتی مقادیر «درست» یا «نادرست» در نظر گرفته می‌شوند
- مثال‌های عملی و اشتباهات رایج

---

## مفهوم اصلی: جهان دو حالتی است

در برنامه‌نویسی، هر سوال پاسخ ساده‌ای دارد: **بله** یا **خیر**، **درست** یا **نادرست**. منطق بولی بنیاد همه تصمیم‌گیری‌ها در کد است.

**مثال: کلیدهای چراغ**
- یک کلید فقط می‌تواند **روشن** یا **خاموش** باشد
- هیچ حالت «شاید» وجود ندارد!
- مقادیر بولی هم همین‌طور کار می‌کنند: **True** (روشن) یا **False** (خاموش)

---

## مقادیر بولی در پایتون

### True و False

```python
# این‌ها فقط دو مقدار بولی هستند
فعال_هست = True
حذف_شده = False

# بررسی نوع
print(type(True))   # <class 'bool'>
print(type(False))  # <class 'bool'>
```

### تبدیل به بولی

```python
# اعداد غیرصفر درست هستند
print(bool(1))      # True
print(bool(-5))     # True (هر غیرصفر!)
print(bool(0))      # False

# رشته‌های غیرخالی درست هستند
print(bool("سلام"))  # True
print(bool(""))       # False (خالی)

# مجموعه‌های غیرخالی درست هستند
print(bool([1, 2, 3]))  # True
print(bool([]))         # False (لیست خالی)
```

## عملگرهای منطقی

### AND: هر دو باید درست باشند

AND را مثل قفل دوکلیده در نظر بگیرید—هر دو کلید باید بچرخند.

```python
# مثال‌های AND
print(True and True)    # True
print(True and False)   # False
print(False and True)   # False
print(False and False)  # False

# واقعی: می‌تواند وارد باشگاه شود؟
بلیط_دارد = True
بزرگسال_هست = True
می‌تواند_وارد_شود = بلیط_دارد and بزرگسال_هست
print(f"می‌تواند وارد شود: {می‌تواند_وارد_شود}")  # True
```

### OR: حداقل یکی باید درست باشد

OR را مثل داشتن دو در در نظر بگیرید—هر کدام شما را داخل می‌برد.

```python
# مثال‌های OR
print(True or True)     # True
print(True or False)    # True
print(False or True)    # True
print(False or False)   # False

# واقعی: آیا تخفیف می‌گیرد؟
دانشجو_هست = False
سالمند_هست = True
tخفیف_می‌گیرد = دانشجو_هست or سالمند_هست
print(f"تخفیف می‌گیرد: {تخفیف_می‌گیرد}")  # True
```

### NOT: معکوس پاسخ

NOT درست را نادرست و نادرست را درست می‌کند.

```python
# مثال‌های NOT
print(not True)   # False
print(not False)  # True

# واقعی: آیا رفتن بیرون ایمن است؟
باران_می‌بارد = False
باید_بیرون_رفت = not باران_می‌بارد
print(f"باید بیرون رفت: {باید_بیرون_رفت}")  # True

# مثال دیگر
وارد_شده = False
if not وارد_شده:
    print("لطفاً وارد شوید!")  # این چاپ می‌شود
```

## جداول درستی

### جدول درستی AND

| A | B | A AND B |
|---|---|---------|
| True | True | **True** |
| True | False | **False** |
| False | True | **False** |
| False | False | **False** |

### جدول درستی OR

| A | B | A OR B |
|---|---|--------|
| True | True | **True** |
| True | False | **True** |
| False | True | **True** |
| False | False | **False** |

### جدول درستی NOT

| A | NOT A |
|---|-------|
| True | **False** |
| False | **True** |

## ترکیب عملگرهای منطقی

### شرایط پیچیده

```python
سن = 25
gواهینامه_دارد = True
هوشیار_هست = True

# هر سه باید درست باشند
می‌تواند_رانندگی_کند = سن >= 18 and گواهینامه_دارد and هوشیار_هست
print(f"می‌تواند رانندگی کند: {می‌تواند_رانندگی_کند}")  # True

# بررسی آخر هفته
روز = "شنبه"
تعطیل_رسمی = False
آخر_هفته_هست = (روز == "شنبه") or (روز == "یکشنبه")
می‌تواند_استراحت_کند = آخر_هفته_هست or تعطیل_رسمی
print(f"می‌تواند استراحت کند: {می‌تواند_استراحت_کند}")  # True
```

### استفاده از NOT با AND/OR

```python
# NOT با AND
ممنوع_هست = False
اجازه_دارد = True
می‌تواند_پست_بگذارد = not ممنوع_هست and اجازه_دارد
print(f"می‌تواند پست بگذارد: {می‌تواند_پست_بگذارد}")  # True

# NOT با OR
خواب_هست = True
غایب_هست = False
در_دسترس_هست = not (خواب_هست or غایب_هست)
print(f"در دسترس هست: {در_دسترس_هست}")  # False
```

## مقادیر درستی (Truthy) و نادرستی (Falsy)

### چه چیزهایی پایتون «درست» می‌داند

این مقادیر در شرایط به عنوان `True` در نظر گرفته می‌شوند:

```python
# اعداد (هر غیرصفر)
if 5:
    print("۵ درست است")  # چاپ می‌شود

# رشته‌های غیرخالی
if "سلام":
    print("رشته غیرخالی درست است")  # چاپ می‌شود

# مجموعه‌های غیرخالی
if [1, 2, 3]:
    print("لیست غیرخالی درست است")  # چاپ می‌شود

if {"کلید": "مقدار"}:
    print("دیکشنری غیرخالی درست است")  # چاپ می‌شود
```

### چه چیزهایی پایتون «نادرست» می‌داند

این مقادیر در شرایط به عنوان `False` در نظر گرفته می‌شوند:

```python
# صفر
if not 0:
    print("۰ نادرست است")  # چاپ می‌شود

# رشته خالی
if not "":
    print("رشته خالی نادرست است")  # چاپ می‌شود

# مجموعه‌های خالی
if not []:
    print("لیست خالی نادرست است")  # چاپ می‌شود

if not {}:
    print("دیکشنری خالی نادرست است")  # چاپ می‌شود

# None
if not None:
    print("None نادرست است")  # چاپ می‌شود
```

### موارد استفاده عملی

```python
# بررسی اینکه آیا کاربر نامی وارد کرده
نام = input("نام خود را وارد کن: ")
if نام:  # True اگر خالی نباشد
    print(f"سلام، {نام}!")
else:
    print("نامی وارد نکردی.")

# بررسی اینکه آیا لیست آیتم دارد
لیست_خرید = ["سیب", "شیر"]
if لیست_خرید:  # True اگر خالی نباشد
    print("باید بری خرید!")
else:
    print("لیستت خالی است.")
```

## اشتباهات رایج مبتدی‌ها

### اشتباه ۱: اشتباه گرفتن `=` با `==`

```python
# ❌ اشتباه
x = 5
if x = 5:  # SyntaxError!
    print("پنج")

# ✅ درست
if x == 5:
    print("پنج")
```

### اشتباه ۲: شرایط بیش از حد پیچیده

```python
# ❌ سخت برای فهم
if not (not بلیط_دارد or not بزرگسال_هست):
    print("می‌تواند وارد شود")

# ✅ ساده‌تر با قانون دمورگان
if بلیط_دارد and بزرگسال_هست:
    print("می‌تواند وارد شود")
```

### اشتباه ۳: مقایسه انواع متفاوت

```python
# ❌ اشتباه
سن = "25"  # رشته
if سن == 25:  # هیچ‌وقت درست نیست!
    print("بزرگسال")

# ✅ درست
سن = int("25")
if سن == 25:
    print("بزرگسال")
```

### اشتباه ۴: استفاده از AND وقتی OR می‌خواهید

```python
# ❌ اشتباه - خیلی محدودکننده
روز = "شنبه"
if روز == "شنبه" and روز == "یکشنبه":  # همیشه False!
    print("آخر هفته")

# ✅ درست
if روز == "شنبه" or روز == "یکشنبه":
    print("آخر هفته")
```

### اشتباه ۵: فراموش کردن اینکه NOT همه چیز بعد از خود را تحت تأثیر قرار می‌دهد

```python
# ❌ گیج‌کننده
آخر_هفته_هست = True
آفتابی_هست = False
if not آخر_هفته_هست and آفتابی_هست:  # یعنی: (not آخر_هفته) AND آفتابی
    print("برو ساحل")  # چاپ نمی‌شود

# ✅ واضح با پرانتز
if not (آخر_هفته_هست and آفتابی_هست):  # یعنی: NOT (آخر هفته AND آفتابی)
    print("شاید بمون خونه")  # چاپ می‌شود
```

### تمرین ۲: سیستم کنترل دسترسی

برنامه‌ای بنویسید که تصمیم بگیرد آیا کسی می‌تواند به فایل دسترسی پیدا کند:

```python
# اطلاعات کاربر
وارد_شده = True
مدیر_هست = False
فایل_عمومی_هست = True

# دسترسی اگر: (وارد شده AND مدیر است) OR فایل عمومی است
dسترسی_دارد = (وارد_شده and مدیر_هست) or فایل_عمومی_هست

if دسترسی_دارد:
    print("دسترسی مجاز!")
else:
    print("دسترسی رد شد!")

# مقادیر بالا را تغییر دهید تا سناریوهای مختلف را تست کنید
```

### تمرین ۳: بررسی صلاحیت

بررسی کنید آیا کسی واجد شرایط تخفیف دانشجویی است:

```python
سن = int(input("سن خود را وارد کن: "))
کارت_دانشجویی_دارد = input("آیا کارت دانشجویی دارید؟ (بله/خیر): ").lower() == "بله"

# واجد شرایط اگر زیر ۲۵ با کارت OR زیر ۱۸
واجد_شرایط_هست = (سن < 25 and کارت_دانشجویی_دارد) or (سن < 18)

if واجد_شرایط_هست:
    print("شما واجد شرایط تخفیف دانشجویی هستید!")
else:
    print("متأسفانه شما واجد شرایط نیستید.")
```

### تمرین ۴: رفع اشکالات

خطاهای منطقی را پیدا و رفع کنید:

```python
# برنامه با اشکال

    print("دسترسی ویژه")
```

## مرجع سریع

### عملگرهای منطقی

| عملگر | کار | مثال | نتیجه |
|--------|-----|------|--------|
| `and` | هر دو باید درست باشند | `True and False` | `False` |
| `or` | حداقل یکی درست | `True or False` | `True` |
| `not` | معکوس | `not True` | `False` |

### مقادیر درستی (Truthy)

| مقدار | نتیجه بولی |
|-------|------------|
| `1, 5, -10` (هر غیرصفر) | `True` |
| `"سلام"` (رشته غیرخالی) | `True` |
| `[1, 2]` (لیست غیرخالی) | `True` |
| `{"کلید": "مقدار"}` (دیکشنری غیرخالی) | `True` |

### مقادیر نادرستی (Falsy)

| مقدار | نتیجه بولی |
|-------|------------|
| `0` | `False` |
| `""` (رشته خالی) | `False` |
| `[]` (لیست خالی) | `False` |
| `{}` (دیکشنری خالی) | `False` |
| `None` | `False` |

### تقدم عملگر

۱. `not` (بالاترین)
۲. `and`
۳. `or` (پایین‌ترین)

**مثال:** `not a and b or c` یعنی `((not a) and b) or c`

---

## نکات کلیدی

۱. **مقادیر بولی** فقط `True` یا `False` هستند
۲. **AND** نیاز دارد هر دو شرط درست باشند
۳. **OR** نیاز دارد حداقل یک شرط درست باشد
۴. **NOT** درست را نادرست و بالعکس می‌کند
۵. **مقادیر درستی** (غیرصفر، غیرخالی) به عنوان True در نظر گرفته می‌شوند
۶. **مقادیر نادرستی** (صفر، خالی، None) به عنوان False در نظر گرفته می‌شوند
۷. **از پرانتز** برای واضح کردن منطق پیچیده استفاده کنید

## گام بعدی

حالا که منطق بولی را درک کردید:
- درباره شرایط تو در تو یاد می‌گیرید (شرایط داخل شرایط)
- منطق بولی را با حلقه‌ها ترکیب می‌کنید
- برنامه‌های تصمیم‌گیری پیچیده‌تر می‌نویسید
