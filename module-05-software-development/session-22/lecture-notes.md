## III. Capstone Project Planning (30 minutes)

### Project Requirements
**Must Include:**
- Multiple functions with proper documentation
- User input and output
- Data storage (files or collections)
- Error handling
- Modular code organization
- Complete user interface

**Project Options:**

#### Option 1: Personal Finance Tracker
```
Features:
- Add/remove income and expenses
- Calculate balance and spending categories
- Generate monthly reports
- Save data to file
- Budget alerts and warnings
```

#### Option 2: Task Management System
```
Features:
- Create, edit, delete tasks
- Set priorities and due dates
- Mark tasks as complete
- Filter and search tasks
- Save/load task lists
```

#### Option 3: Simple Quiz/Game Application
```
Features:
- Multiple choice or fill-in questions
- Score tracking and feedback
- Question categories
- Save high scores
- Admin interface to add questions
```

#### Option 4: Student Choose: Custom Project
```
Requirements:
- Approved by instructor
- Demonstrates course concepts
- Solves a real problem
- Complete and functional
```

### Project Planning Framework
```
1. Problem Definition
   - What problem does your project solve?
   - Who will use it?

2. Feature List
   - Core features (must-have)
   - Nice-to-have features
   - Future enhancements

3. Data Design
   - What data do you need to store?
   - How will you structure it?

4. User Interface Design
   - How will users interact with your program?
   - What menus/options will you provide?

5. Implementation Plan
   - What modules/functions do you need?
   - What order will you build features?
```

### Example Project Plan: Task Manager
```
Problem: People need to organize and track their tasks
Users: Students, professionals, anyone with tasks

Core Features:
- Add new tasks with description and priority
- List all tasks with status
- Mark tasks as complete
- Delete completed tasks

Data Structure:
tasks = [
    {"id": 1, "description": "Buy groceries", "priority": "high", "completed": False},
    {"id": 2, "description": "Study for exam", "priority": "medium", "completed": False}
]

User Interface:
1. Add task
2. List tasks
3. Complete task
4. Delete task
5. Save and exit

Implementation Steps:
1. Create task data structure and basic functions
2. Build user interface menu
3. Implement add/list/complete/delete functions
4. Add file save/load functionality
5. Add error handling and validation
```

## IV. Implementation and Support (30 minutes)

### Coding Time
- Students work on their capstone projects
- Instructor circulates to provide help
- Focus on applying all learned concepts

### Common Issues and Solutions
```
Problem: "I don't know where to start"
Solution: Break into small functions, start with data structures

Problem: "My code is getting messy"
Solution: Create separate modules, use functions for organization

Problem: "Users can break my program"
Solution: Add input validation and error handling

Problem: "I can't save/load data"
Solution: Review file I/O examples, start simple
```

### Progress Checkpoints
- **20 minutes**: Basic data structure and one function working
- **40 minutes**: Core features implemented
- **60 minutes**: Full functionality with error handling
- **Final**: Polished interface and documentation

## V. Project Presentation and Assessment (20 minutes)

### Presentation Format
Each student presents their project (3-5 minutes):
```
1. Problem Statement (30 seconds)
   - What does your program do?

2. Demo (2 minutes)
   - Show it working
   - Highlight key features

3. Code Tour (1-2 minutes)
   - Explain one interesting part
   - Show good practices used

4. Challenges & Learning (30 seconds)
   - What was difficult?
   - What did you learn?
```

### Assessment Criteria
```
Functionality (40%):
- Program works as intended
- All required features implemented
- Error handling works

Code Quality (30%):
- Modular organization
- Proper functions and documentation
- Good variable names and structure

User Experience (20%):
- Clear interface
- Helpful error messages
- Intuitive navigation

Creativity & Effort (10%):
- Original features
- Polished presentation
- Extra effort shown
```
